# Architecture

## Principles

- Modular components.
- Reusable utilities.
- Client-side whenever possible.
- Deterministic logic.
- Minimal dependencies.
- Responsive UI.
- Accessible UI.

## Component Philosophy

Extract reusable logic instead of duplicating code.

Avoid large page files.

Separate:

- UI
- Utilities
- Business logic

## Refactoring

Improve architecture gradually.

Avoid massive rewrites.

Each feature should leave the codebase cleaner than before.
